# Portfolio Positioning

## Aoki Design Studio の役割定義

- Aoki Design Studio は、制作物そのものの見た目だけでなく、見積・相談・比較検討まで含めた導線設計を見せるためのポートフォリオ文脈です。
- 作品ごとに「何を作ったか」だけでなく、「どの判断を助ける設計か」を説明できる状態を目指します。
- 過度に営業色を強めるよりも、UI 設計、情報整理、相談前の体験設計を落ち着いた言葉で伝えることを優先します。

## Works Finder の役割定義

- Works Finder は、ポートフォリオ内で検索・比較・検討支援 UI 設計を担う作品です。
- 単なる一覧ページではなく、近い案件を探し、比較し、判断材料を持ち帰るところまでをひとつの体験として見せます。
- 個別作品ページが案件単位の説明を担うのに対し、Works Finder は横断的な探索体験と状態設計を担います。

## ポートフォリオトップに載せる想定の短文

- 制作実績を探して比較できる、検討支援 UI として組んだポートフォリオアーカイブです。
- 一覧を見せるだけで終わらせず、近い案件を探し、比べ、相談準備までつなぐ UI を設計しています。
- 検索・比較・相談導線までを一連で見せる、掲載用アーカイブ UI の検証作品です。

## 作品紹介文の短文

- 制作実績を検索・比較しやすい一覧体験へ再構成した Works Finder。
- 近い案件を探し、判断材料を残しやすくするための検討支援 UI。
- ポートフォリオ内の案件を横断的に見比べられる、検索・比較アーカイブ。

## 依頼者目線へ翻訳した価値訴求の書き換え例

- 書き換え前:
  実績一覧を検索しやすくしました。
- 書き換え後:
  近い依頼事例を探しやすくし、比較しながら相談内容を整理しやすい UI として設計しています。

- 書き換え前:
  比較機能を付けています。
- 書き換え後:
  候補を並べて違いを確認できるため、検討段階での迷いを減らしやすい構成です。

- 書き換え前:
  相談導線を用意しています。
- 書き換え後:
  気になった案件を見た流れのまま、相談前の要点をまとめやすい導線を用意しています。

- 書き換え前:
  一覧ページを作りました。
- 書き換え後:
  案件単位の魅力だけでなく、複数案件を見比べる体験まで含めて設計したアーカイブ UI です。

## 使わない方がいい表現

- 実績が爆増した
- 問い合わせが大幅改善した
- CV を最大化した
- 圧倒的に成果が出た
- 業界トップレベル
- 実在案件かどうか判別しにくいまま「クライアント成果」と読める表現

補足:
成果を断定する数値や、確認できていないクライアント効果は書かず、必要な場合は「狙い」「想定効果」「判断しやすくするための設計」のように保守的に言い換えます。

## 今後メインポートフォリオ側へ転記する前提の文言テンプレ

### トップ導入用

- `{作品名}` は、`{役割}` を担う作品です。`{主要行為1}` から `{主要行為2}` までを一連で設計しています。
- `{作品名}` は、`{対象}` が `{判断}` しやすくなるように構成した `{作品種別}` です。
- `{作品名}` は、`{一覧 / 検索 / 比較 / 相談}` のどこを担う作品かを明確にした掲載用デモです。

### 作品紹介用

- `{作品名}` は、`{主要機能}` を通して `{見せたい力}` を説明するための作品です。
- `{作品名}` では、`{課題}` を `{設計方針}` で整理しています。
- `{作品名}` は、`{誰向け}` に対して `{何を分かりやすくするか}` を重視した構成です。

### Works Finder 用の埋め込み例

- Works Finder は、検索・比較・検討支援 UI 設計を担う作品です。
- ポートフォリオ内の案件を横断して探しやすくするための、掲載用アーカイブ UI です。
- 個別案件の魅力を見せるだけでなく、比較検討の体験まで設計した作品です。

### 個別作品用の埋め込み例

- この作品は、見積・相談導線設計をどのように組み立てるかを見せるための詳細ページです。
- このページでは、案件単位での情報設計と相談導線の考え方を補足しています。
- 一覧側が比較検討を担うのに対し、このページは個別案件の判断材料を補う役割です。
